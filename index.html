Big Data Security Challenges

  • Cross Domain Security



    Data must be secure at all times, even when moving from one security domain to another.

  • Polyglot & Dynamic Environment


    Evolving infrastructure: newly added machines must be secured as we go.
    A polyglot environment implies multiple security measures

  • Fine-grained Access Management


    Each data item must have its own security policy, as opposed to providing a single policy for complex aggregated structures.

  • Data Ownership



    A data governance process must be set to define data ownership/stewardship.

  • Privacy & Data Anonymization


    Privacy constraints such as GDPR must be taken into consideration.
    Colocalization of data and processing requires a certain level of opacity.

  • Data provenance & Lineage


    As data comes from several sources and is subject to multiple transformations, it must be tracked throughout its lifecycle.

And this is in addition to classical security measures

Zero Trust Model

  • Assume that there is no way to keep intruders out.

  • Assume that everyone on the inside of the organization is a threat to sensitive data.

Data Centric Security

THE HOW?

  • Data Discovery

    Using patterns, regular expressions or dictionaries to detect sensitive data.

  • Classification

    Tagging files with metadata to indicate their content and appropriate use.

  • Persistent Encryption

    Make sure the data is accessed by the appropriate users at all times.

  • Data Masking & Tokenization

    Replacing sensitive data with non sensitive data, or completely removing it.

A few Big Data Security Solutions...

Hadoop Encryption Mechanisms

Wire Encryption (Data in Motion)

  • Client-NameNode Communication: RPC with SASL
  • NameNode-DataNode Communication: Data Transfer Protocol with 3DES or RC4
  • UI-NameNode/DataNode Communication: HTTP with SSL

Transparent Data Encryption (Data at Rest)

  • HDFS-level encryption
  • Transparent to the applications

Apache Knox Gateway

  • Gateway providing perimeter security
  • Encapsulates Kerberos while simplifying its configuration
  • Knox intercepts REST/HTTP calls and provides authentication, authorization, audit, URL rewriting, web vulnerability removal and other security services

Apache Ranger

  • Centralization of the authorization management across all Hadoop components (HDFS, Hive, etc.)
  • Centralization of log files for audit
  • Provides support for fine-grained authorization
  • Enhanced support for attribute-based access control

Overall Secure Hadoop Architecture

Voltage Secure Data

  • Data-centric approach to security for Big Data and Cloud systems
  • Secures the data from unauthorized access in motion, at rest, and in use
  • De-identifies the data as close to its source as possible
  • Fine-grained security through tokenization, encryption, masking
